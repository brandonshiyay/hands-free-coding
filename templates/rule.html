Variables assigning:

[Variable Type] [Variable Name] is/equals [Variable Value]


Variable operations:

[Variable Name] is/equals [Variable Operation]

Function declaring:

Define function [Function Name] parameters [Parameters]

Close function - End function declaration

* Function_name can be multiple words, as long as it comes before "parameter(s)" (i.e. "my_function")

* Parameters can be multiple variables (i.e. "x y z") or none at all (don't say anything)


Function calls:

Call function [Function Name] arguments ["Variable" Variable Name] / [Strings]

* Function_name can be multiple words, as long as it comes before "argument(s)" (i.e. "my_function")

* Function calls can take in a mix of variables and only strings as arguments (i.e. "call function my_print variable x hello world variable y" is equivalent to my_print(x, "hello world", y))


Print function call:

Print / Prince ["Variable" Variable Name] / [Strings]

* Print can print out a mix of variables and strings (i.e. "print variable x hello world variable y" is equivalent to print(x, "hello world", y))

* "prince" is there for leniency

If statements:

(Else / Elf) If ["Variable" Variable Name] (is / equals to / is not / does not equal to / is greater than / is less than) [Boolean / Integer]

Else

Close if - close all if statements

* (El)if only supports comparisons between variables or between variables and a int/bool
* "Else" is only needed to be said in order to instantiate the else statment
* Remember to close each if/elif/else by saying "close if"
* "Elf" is there for leniency
Main Function

In Main
* "In main" automatically sends the user into the "if __name__ == '__main__'" function

Undo / Redo:

"undo" - undos the last command
"redo" - redos the last undo
* Able to say either indefinitely without repercussions

Tab In / Out:

"tab in" - reduces the number of tabs by 1
"tab out" - increases the number of tabs by 1
